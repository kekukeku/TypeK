# Product Brief: SayIt

## Executive Summary

SayIt 是一款跨平台桌面語音輸入工具，目標是讓使用者在任何應用程式中透過一個按鍵即可將語音轉為整理過的繁體中文書面文字。產品以「開箱即用」為核心設計原則 — 安裝後設定一組 API Key，按住快捷鍵就能用。轉錄結果經由 AI 自動去除口語贅詞、修正標點、適當分段，直接貼入游標位置。初期為個人效率工具，中期目標擴展至公司內部使用。

---

## Core Vision

### Problem Statement

開發者與知識工作者在日常工作中有大量文字輸入需求（Slack 訊息、PR 描述、會議筆記、文件撰寫等），但打字速度遠跟不上思考速度。語音輸入是自然的解決方案，但現有工具要麼功能複雜需要大量設定（VoiceInk 需管理本地模型、多種提供商），要麼需要付費訂閱（Typeless），要麼辨識品質不足以直接使用（macOS 內建聽寫沒有 AI 後處理，輸出充滿口語贅詞）。

### Problem Impact

- 思考到文字的轉換摩擦導致想法流失或表達品質下降
- 反覆打字造成效率低落，尤其在需要快速回覆或記錄的場景
- 現有語音輸入的原始輸出不夠「乾淨」，使用者仍需手動編輯，抵消了效率增益

### Why Existing Solutions Fall Short

| 方案 | 問題 |
|------|------|
| macOS 內建聽寫 | 無 AI 後處理，口語贅詞保留，標點不準確，無法直接作為書面文字使用 |
| VoiceInk | 功能強大但設定繁複（12+ AI 提供商、本地模型管理、Power Mode），「自己動手組裝」的體驗 |
| Typeless | 需付費訂閱，有免費額度限制 |
| 通用語音轉文字 API | 需要自己寫程式串接，不是終端使用者產品 |

所有現有方案都缺少一個關鍵組合：**零設定的語音輸入 + 自動 AI 書面語整理 + 全域貼上**，三者無縫合一。

### Proposed Solution

一個常駐在系統列的桌面應用程式，核心流程只有一步：按住快捷鍵說話，放開後文字自動出現在游標位置。背後由 Groq Whisper API 完成語音辨識，再經 Groq LLM 以可自訂的 prompt 將口語轉為通順的繁體中文書面語。搭配自訂詞彙字典提升專業術語辨識率，歷史記錄提供使用統計與回顧能力。

### Key Differentiators

1. **極簡設計哲學** — 設定只有三項：快捷鍵、API Key、AI Prompt。不提供不必要的選項，刻意排除語言切換、多提供商、分階段文字處理等複雜功能
2. **AI 即後處理** — 不做多階段文字管線，一次 AI 呼叫搞定去口語、分段、標點全部事情。使用者只需要調整一段 prompt 就能控制所有行為
3. **跨平台開箱即用** — Tauri v2 架構，macOS 和 Windows 各有最佳預設（mac: Fn / win: 右 Alt），不需要使用者理解平台差異
4. **從自用到團隊** — 先為自己打造最順手的工具，驗證後可推廣至公司內部，零學習成本

**Persona 2：Mia — 產品經理**

- **現有方案：** 沒有用過桌面語音輸入工具
- **期望：** 零學習成本，按住按鍵說話就好，不需要理解技術細節
- **成功指標：** 每天的文字輸入工作量體感減少，回覆客戶的速度變快

## Success Metrics

### User Success Metrics

| 指標 | 目標 | 衡量方式 |
|------|------|---------|
| **輸出可用率** | > 90% 的轉錄結果不需要手動修改即可直接使用 | 使用者主觀回饋 + 觀察是否觸發 undo/修改 |
| **日活使用率** | 部署後 > 80% 的使用者每個工作日都會使用 | Dashboard 歷史記錄（每日有記錄的使用者數） |
| **端到端延遲** | 放開按鍵到文字出現 < 3 秒（含 AI 整理） | 歷史記錄中 `transcriptionDurationMs + enhancementDurationMs` |
| **首次使用時間** | 從安裝到第一次成功語音輸入 < 2 分鐘 | Onboarding 流程觀察 |

### Business Objectives

純內部效率工具，不涉及營收指標。核心目標：

1. **提升團隊文字輸入效率** — 減少打字時間，讓團隊成員把時間花在思考而非輸入
2. **降低導入門檻** — 非技術角色也能零學習成本上手，不需要 IT 支援
3. **穩定可靠** — 作為日常工具，穩定性比功能豐富度重要

### Key Performance Indicators

| KPI | 目標值 | 時間框架 |
|-----|--------|---------|
| 日均使用次數（每人） | >= 10 次 | 部署後 1 個月 |
| 團隊採用率 | > 80% 日活 | 部署後 2 週 |
| AI 整理輸出滿意度 | 不需修改率 > 90% | 持續追蹤 |
| 累計節省時間 | Dashboard 可見 | 持續累計 |
| 系統可用率 | > 99%（排除網路問題） | 持續追蹤 |

## MVP Scope

### Core Features

| # | 功能 | 說明 |
|---|------|------|
| 1 | **跨平台熱鍵系統** | `rdev` crate 統一監聽，macOS 預設 Fn、Windows 預設右 Alt，可自選修飾鍵，支援 Hold + Toggle 雙模式 |
| 2 | **AI 文字整理** | 轉錄文字送 Groq LLM + 可編輯 prompt，一次完成去口語、分段、標點修正。字數 < 10 字跳過 AI 直接貼上 |
| 3 | **自訂詞彙字典** | CRUD 管理常用詞彙，注入 Whisper API prompt 提升辨識率，同時注入 AI 上下文 |
| 4 | **歷史記錄** | SQLite 持久化每筆轉錄（原始文字、整理後文字、時長、字數等），供 Dashboard 統計與回顧 |
| 5 | **App UI — Dashboard** | 統計卡片（總口述時間 / 字數 / 節省時間 / 速度 / 使用次數 / AI 使用率）+ 最近轉錄列表 |
| 6 | **App UI — 歷史 / 字典 / 設定** | 歷史記錄搜尋與複製、詞彙 CRUD、快捷鍵設定 / Groq API Key / AI Prompt 編輯 |
| 7 | **HUD 狀態擴展** | 新增 `enhancing` 狀態，完整狀態機：idle → recording → transcribing → enhancing → success/error → idle |

### Out of Scope for MVP

| 項目 | 原因 | 預計階段 |
|------|------|---------|
| VAD 靜音偵測 | 需 Web Audio API 研究，Toggle 模式先以手動停止為主 | Phase 2 |
| 串流轉錄（即時字幕） | 需 WebSocket 整合，核心體驗已夠用 | Phase 2 |
| Mini HUD 模式 | 外接螢幕適配，優先級低 | Phase 2 |
| 剪貼簿還原 | 目前不還原已確認可接受 | Phase 2 |
| 多 AI 提供商 | 僅 Groq，避免過度設計 | 未定 |
| 多語言切換 | 輸出統一繁體中文 | 未定 |
| Per-App 設定（Power Mode） | VoiceInk 風格，複雜度高 | 未定 |
| IT 統一部署管理 | 先個人安裝模式 | 未定 |

### MVP Success Criteria

MVP 達成以下條件即視為成功，可進入 Phase 2：

1. **核心流程可用** — 按鍵 → 錄音 → 轉錄 → AI 整理 → 貼上，端到端延遲 < 3 秒
2. **跨平台運作** — macOS 和 Windows 上都能正常使用
3. **輸出品質** — > 90% 的轉錄結果可直接使用，不需要手動修改
4. **團隊可部署** — 公司同事能在 2 分鐘內完成安裝並開始使用
5. **資料可追蹤** — Dashboard 正確顯示使用統計

### Future Vision

**Phase 2 — 體驗優化：**
- VAD 靜音偵測搭配 Toggle 模式，實現真正的免手動語音輸入
- 串流轉錄即時字幕，邊說邊看
- Mini HUD 適配外接螢幕

**長期方向：**
- Per-App 智慧設定（不同 App 自動切換 prompt）
- 多語言支援（英文、日文等）
- IT 集中管理（API Key 統一配置、使用量監控）
- 考量是否開源或轉為公司產品
